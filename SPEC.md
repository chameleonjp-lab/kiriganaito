# kiriganaito 仕様書

## 1. ゲーム概要

`kiriganaito` は、左へ走っているように見える 🚚 をタップでジャンプさせ、穴・でこぼこ道・通行人・車両を避けながら補正込み走行距離を伸ばすスマホ向けブラウザゲームです。

- 公開予定 URL: `https://chameleonjp.codeberg.page/kiriganaito/`
- ゲーム slug: `kiriganaito`
- 表示名: `kiriganaito`
- 想定端末: iPhone SE 相当の小さいスマホ縦画面
- プレイ時間目安: 標準 45〜90 秒、上級者は 2〜3 分以上
- 実装形態: `index.html` だけで完結する HTML/CSS/JavaScript 単体ファイル

## 2. 固定仕様

- 外部画像、外部音声、外部フォント、npm、Vite、ビルド環境は使いません。
- ゲーム中に大量に動く要素は Canvas に描画します。
- 障害物、アイテム、パーティクルを DOM 要素として大量配置しません。
- 横スクロールを出しません。
- 物理演算、当たり判定、距離加算、逃走時間、取り逃がし判定は `1/60` 秒の固定タイムステップで進めます。処理落ち時は 1 フレーム最大 5 ステップまでに制限し、過剰な蓄積時間は捨てます。
- 結果画面に入ったら、ゲーム更新、敵移動、タイマー、当たり判定、スコア加算を止めます。
- ランキング送信はゲーム終了時に自動で 1 回だけ行います。
- 結果画面にランキング登録ボタンは置きません。
- Supabase で使ってよいキーは Publishable key のみです。
- `public.scores` は使いません。
- secret key、service_role key、管理者用キーは書きません。

## 3. 勝利条件、敗北条件

固定ゴールはありません。1 回のプレイで補正込み走行距離をできるだけ伸ばすことが目的です。

敗北・終了条件は次の 3 つです。

1. 穴に落ちる
   - 即終了。
   - 事故扱いではありません。
   - 無敵時間中でも即終了します。
2. 警戒度が 3 になる
   - 障害物にぶつかるたびに警戒度が 1 上がります。
   - 警戒度 3 で 🚓 に捕まって終了します。
3. リタイアする
   - 上部のリタイアボタンで終了します。
   - その時点の補正込み走行距離を結果として扱います。
   - ランキング送信も 1 回だけ行います。

## 4. 操作方法

- Canvas をタップ、または下部の大きいジャンプボタンをタップしてジャンプします。
- 2 段ジャンプはありません。
- コヨーテタイム: 地面から離れた後 `0.08` 秒までジャンプを受け付けます。
- ジャンプバッファ: 着地直前タップを `0.12` 秒まで保存します。
- 衝突後の障害物判定無敵時間は `0.7` 秒です。
- 入力は `pointerdown` 中心で扱い、タッチ・マウス・ペンをまとめます。
- タップに成功してジャンプが成立した瞬間だけ、小さいジャンプ開始煙を Canvas パーティクルで出します。ジャンプできない状態では煙を出しません。
- 着地時は小さい着地煙と短い着地沈み込みを出し、地面に戻ったことを視覚的に分かりやすくします。
- ジャンプボタンは `pointerdown` 時に短く沈む表示になります。これは操作感の演出だけで、ジャンプ回数や判定は変えません。
- 結果画面ではジャンプ入力を無効にし、結果画面のボタンだけ反応します。

ジャンプ関連の調整値は `index.html` の定数に分けます。

| 定数 | 値 | 意味 |
| --- | ---: | --- |
| `JUMP_POWER` | `560` | ジャンプ初速 |
| `GRAVITY` | `1450` | 重力加速度 |
| `LANDING_SMOKE_COUNT` | `7` | 着地煙パーティクル数 |
| `JUMP_SMOKE_COUNT` | `4` | ジャンプ開始煙パーティクル数 |
| `SHADOW_MIN_SCALE` | `0.58` | 高く跳んだ時の影の最小倍率 |
| `SHADOW_MAX_SCALE` | `1` | 地面付近の影の最大倍率 |
| `JUMP_APEX_HEIGHT` | `JUMP_POWER * JUMP_POWER / (2 * GRAVITY)` | 影の変化を実ジャンプ最高到達点に合わせるための基準高さ |

## 5. スコア計算

スコア名は「補正込み走行距離」です。走行距離とスコアは意味を分けて扱います。

- 走行距離: 実際に走った距離です。HUD ではプレイヤーが直感的に見たい主表示として `走行 0.00km` 形式で表示します。
- スコア: 走行距離にアイテム加算・事故減点・取り逃がし減点を反映した補正込み走行距離です。HUD では `スコア 0.00km` として走行距離と分けて表示します。
- 単位: km
- 表示形式: `0.00km`
- 内部送信値: 整数メートル
- 高い方がよいベストスコア型
- 結果画面の最終スコアとランキング送信値は、従来通り補正込みスコアを使います。結果画面の大きなスコア、`resultSnapshot.score`、ランキング RPC の `p_score` は必ず一致させます。

計算式:

```text
補正込み走行距離 = 実走行距離 + アイテム加算 - 事故ペナルティ - 取り逃がしペナルティ
```

- 💰 は取得で `+0.10km`、内部値 `+100m` です。取り逃がしても減点しません。
- 🔩 は取得で `+0.03km`、内部値 `+30m` です。取り逃がすと `-0.05km`、内部値 `-50m` です。
- ⚙️ は取得で `+0.07km`、内部値 `+70m` です。取り逃がすと `-0.05km`、内部値 `-50m` です。
- 👯‍♀️ は取得で 4 秒間、障害物事故に対して無敵になります。スコア加算はありません。取り逃がしても減点しません。
- 事故ペナルティ: 1.00km 未満は 1 回 `-0.10km`（内部値 `-100m`）、1.00km 以降は 1 回 `-0.20km`（内部値 `-200m`）
- 表示値と送信値は 0 未満にしません。
- 逃走中も距離加算は通常のままです。見た目や接近速度だけ 2 倍になります。
- 見た目専用スクロールを速くしても、実走行距離や補正込みスコアの加算速度は変えません。
- 結果確定後にスコアは変えません。
- プレイ中は `run.maxRunMeters` / `run.lastNonZeroRunMeters` 相当の最大実走行距離を保持し、穴落ちやリタイアの直前に `run.runMeters` が古い値や 0 に戻っても、実際に走った最大距離を最終実走行距離として結果へ固定します。`run.elapsed > 1` 秒なのに結果スコアが 0 のままになる状態は異常としてテストで検出します。

## 6. 障害物仕様

障害物は Canvas 上で絵文字として描きます。内部では `direction` で進行方向を管理します。

| 種類 | 速度 |
| --- | ---: |
| 🛵 | 3.6 |
| 🚗 | 3.8 |
| 🏃🏻 | 2.0 |
| 🚴 | 3.0 |
| 🚶 | 1.0 |

- 左向き障害物: 進行方向側の障害物です。自走する相手ではなく、道路上にいる静止物として扱います。画面上では道路スクロールに乗って流れるだけです。
- 右向き障害物: 🚚 と逆方向へ動く相手で、正面から向かってくるため接近が速い見た目です。描画時は左右反転します。
- 進行方向側の障害物は `STATIC_OBSTACLE_SCROLL_PX` を使い、道路・穴・通常アイテムと近い通過速度にします。仕様上の速度値は対向障害物の接近速度や将来調整用として残します。
- 🚴 は 1 回ジャンプで越えられるよう、静止物としてスクロールさせることに加えて上側の当たり判定を見た目と大きくズレない範囲で数 px だけ低くします。🚴 を無害化したり、すり抜けにしたりはしません。
- 対向障害物のみ正面から向かってくる高い接近速度を使います。
- 同じ障害物で連続減点しないよう、衝突した障害物は非アクティブ化します。
- 距離に応じて、🚶、🏃🏻、🚴、🛵、🚗、対向障害物の順で増やします。今回の緊急調整では障害物予定間隔を従来比おおむね 0.33〜0.45 倍へ短縮し、60 秒相当の自然走行で障害物出現数が約 3 倍になることを狙います。


## 6.1 出現ロジックと難易度上昇

出現は毎フレーム乱数だけに頼らず、距離ベースの予定値で管理します。通常走行の `BASE_SPEED` は `4.35` を維持し、🚚の体感速度は前回調整より少し速い状態を維持します。背景の `VISUAL_SCROLL_SPEED` は `200` で、ランキング上は同じスコア式のままです。

- `nextObstacleAt`、`nextHoleAt`、`nextItemAt` で次の出現距離を管理します。初期予定は `0.07km`、`0.13km`、`0.10km` として開始直後の待ち時間を短くします。
- `lastDangerAt`、`lastHoleAt`、`lastHoleKind`、`lastObstacleAt`、`lastOncomingAt` で直近の危険配置を記録します。
- 逃走中は見た目速度が 2 倍になるため、何もない時間を減らしつつ、穴直後・対向障害物直後の最低限の逃げ場は維持します。
- `nextOncomingAt` で対向障害物候補の距離も管理します。3.00km 未満は出さず、3.00km 到達後の初回は約 `0.05〜0.12km`、3.00〜5.00km は約 `0.12〜0.22km`、5.00km 以降は約 `0.09〜0.18km`、逃走中は約 `0.07〜0.14km` ごとに候補を作り、安全距離が足りなければ延期します。対向障害物の選択重みも上げますが、視認不能な速度や大穴直後の対向は避けます。
- `spawnStuff()` は直接巨大化させず、`selectPatternByDifficulty(km, d)` と `spawnPattern(pattern, km, d)` で短い配置パターンを選んでから生成します。
- パターン名は DEBUG 時に表示できます。現在の主なパターンは `short_hole`、`small_hole`、`medium_hole`、`large_hole`、`walker`、`hole_then_bike`、`high_coin`、`risky_gear`、`oncoming_safe`、`breather`、`dancer`、`small_hole_cash`、`cash_before_oncoming`、`cash_after_oncoming`、`bolt_before_small_hole`、`medium_hole_gear`、`chase_small_hole`、`chase_oncoming` です。歩行者から固定の報酬へつなぐ旧セットは削除済みのままです。
- `3.00km` 未満では対向障害物を出しません。
- 0.50km 以降から障害物・穴・アイテム・対向候補の出現間隔を短くして空白時間を減らします。今回の緊急調整では穴予定間隔を従来比おおむね 0.5〜0.65 倍へ短縮し、小穴系パターンの重みを増やして穴出現数を約 2 倍にします。3.00km 以降はさらに密度を上げ、常に次の判断が見えやすいテンポにします。ただし、穴同士、穴と障害物、対向障害物同士、アイテムと危険物の最低距離は維持し、避けられない配置は禁止します。
- 逃走中は `breather` パターンの比率を下げ、`chase_small_hole` と `chase_oncoming` を増やします。`risky_gear` は自然選択しません。ランダムアイテムも 🔩・⚙️ のような取り逃がしペナルティ付きアイテムを抑え、逃走中の主目的を「生き残る」ことにします。👯‍♀️ は 1.50km 未満では出現させず、出現後は最低 1.50km の距離クールダウンを置きます。`dancer` 重みは 0.04 に抑え、`breather` からは原則 👯‍♀️ を出しません。

| 距離 | 内容 |
| --- | --- |
| 0.00〜0.50km | 練習区間。短い穴、🚶中心。対向障害物なし。 |
| 0.50〜1.50km | 小穴、🚶、🏃🏻、🚴、少量アイテムで軽い判断を増やし、約 1.0〜1.8 秒ごとに何かを見せる。 |
| 1.50〜3.00km | 小穴・中穴・静止障害物で密度を上げ、約 0.9〜1.5 秒ごとに判断を作る。 |
| 3.00〜5.00km | 小穴・静止障害物・対向障害物・リスク報酬をさらに増やし、ほぼ常に次の判断が見えている状態にする。ただし連続対向は禁止。 |
| 5.00km以降 | 高密度区間。対向障害物と穴を組み合わせるが、1回ジャンプで対応不能な配置は禁止。 |

最低間隔の実装値は次の通りです。単位は km です。

| 制限 | 値 |
| --- | ---: |
| 障害物直後に穴を出さない距離 | `0.12` |
| 穴直後に障害物を出さない距離 | `0.18` |
| 障害物同士の最低距離 | `0.14` |
| 対向障害物後の安全距離 | `0.20` |
| アイテムと障害物の最低距離 | `0.07` |
| アイテムと穴の最低距離 | `0.06` |

穴は `lastHoleKind` と既存 `holes` 配列の画面上位置でも判定します。穴同士は接触させず、実質 1 つの長すぎる穴になる配置を禁止します。小穴の連続は許可しますが、必ず着地できる短い地面を挟みます。中穴直後の中穴・大穴、大穴直後の穴や対向障害物は延期し、2段ジャンプが必要な穴配置は禁止します。穴中央の ⚠️ 表示と穴即終了仕様は維持します。

| 穴同士の基本最低距離 | `0.14` |
| 小穴同士・小穴直後の最低距離 | `0.10` |
| 中穴を含む穴同士の最低距離 | `0.14` |
| 大穴を含む穴同士の最低距離 | `0.20` |
| 同時表示中の穴同士の画面上最低地面 | `54px` 以上 |

## 7. 事故・逃走モード仕様

障害物にぶつかると事故扱いになります。

- 補正込み走行距離から `-0.20km`、内部値 `-200m`
- 画面上部で 🚨 が光ります。
- 🚓 が 🚚 を追いかけます。事故回数が増えるほど、Canvas 上の 🚓 は少し近く見えるように描きます。
- 15 秒間、逃走モードになります。
- 逃走モード中は見た目と障害物接近速度を 2 倍にします。
- 逃走モード中は背景色を少し緊急感のある色にし、赤い帯と「逃げろ」文字を Canvas 内に流します。「逃げろ」文字は軽く揺れます。赤い帯と文字は障害物、穴、アイテムの視認性を優先して控えめな透明度にします。
- 逃走残り `3` 秒以下では 🚨 と HUD の残り秒表示を少し強く点滅させます。
- 逃走終了時は約 `1.15` 秒だけ「逃げ切り」と表示します。表示だけで、スコアボーナスはありません。
- 警戒度を 1 上げます。
- 逃走中に再事故した場合も同じ処理を行い、逃走残り時間は 15 秒に戻します。
- 警戒度が 3 になったら即終了します。
- 逃走が終わっても警戒度は下がりません。

## 8. 道、穴、でこぼこ仕様

- 道路は Canvas に地面色、黄色い地面ライン、控えめな白い短線を描き、左へ進んでいる速度感を出します。
- スコア計算用の `run.runMeters` とは別に、見た目専用スクロール `run.visualScroll` を持ちます。道路線、背景、街明かり、星、地面の流れはこの見た目専用値で動かし、🚚 が走っている速度感を強めます。
- 見た目専用スクロールは逃走中にさらに速くなりますが、逃走中の距離加算は通常のままで、スコア計算やランキング送信値には影響しません。
- 🚚 の下には楕円の影を Canvas で描きます。地面に近いほど濃く大きく、高いほど薄く小さくします。影のスケールは `JUMP_POWER` と `GRAVITY` から求める実ジャンプ最高到達点に合わせます。影は当たり判定に使いません。
- 穴はジャンプで越える必要があります。
- 穴に落ちたら即終了します。
- 穴幅は 1 回ジャンプで越えられる範囲に制限します。
- 2 段ジャンプが必要な穴幅は出しません。
- でこぼこは直接ダメージにせず、地面の高さの軽い上下と見た目演出に使います。
- 見た目が荒れていても当たり判定の地面はなだらかにします。
- 穴直後に避けられない障害物を置かず、最低間隔を持たせます。
- 穴は黒い四角だけでなく、暗いグラデーション、縁取り、手前側のハイライトを描いて、穴だとすぐ分かるようにします。
- 穴の描画は `ctx.save()` / `ctx.restore()` で閉じ、`globalAlpha`、`shadowBlur`、`filter`、`globalCompositeOperation` などの描画状態が 🚚 の描画へ漏れないようにします。穴の手前で 🚚 が黒く見えないよう、🚚 は穴より後に通常状態で描きます。
- 穴の中央付近には視認性向上のため `⚠️` を Canvas で表示します。`⚠️` は表示専用で、穴・プレイヤー・障害物の当たり判定には使いません。
- 穴の見た目と当たり判定は大きくズレない範囲に保ちます。


### 8.1 穴サイズ分類

穴は役割別に小穴・中穴・大穴へ分けます。穴には従来通り落ち、穴中央付近の `⚠️` 表示も維持します。

| 種別 | 幅 | 役割 | 追加安全距離 |
| --- | ---: | --- | ---: |
| 小穴 | `30〜44px` | 序盤から出すテンポ作り。3km以降に増やす。 | `0.07km` |
| 中穴 | `48〜66px` | 0.50km以降の基本ジャンプ障害。 | `0.11km` |
| 大穴 | `70〜86px` | 5km以降にたまに出す緊張要素。連続出現と直後の対向障害物は禁止。 | `0.16km` |

2段ジャンプが必要な幅は出さず、大穴を難易度上昇の主役にはしません。テンポ改善の主役は小穴です。

## 9. アイテム仕様

拾うと補正込み走行距離に加算または一時効果を付与します。従来のコイン表示は 💰 に変更し、画面表示と仕様書では 💰 として扱います。

| アイテム | 取得時の表示 | 内部値 | 取り逃がし |
| --- | ---: | ---: | ---: |
| 💰 | +0.10km | +100m | なし |
| 🔩 | +0.03km | +30m | -0.05km / -50m |
| ⚙️ | +0.07km | +70m | -0.05km / -50m |
| 👯‍♀️ | スコア加算なし | 4秒間の障害物事故無敵 | なし |

- 💰、🔩、⚙️ は単なるおまけではなくリスク報酬として再配置します。`small_hole_cash`、`cash_before_oncoming`、`cash_after_oncoming`、`bolt_before_small_hole`、`medium_hole_gear` のように「取ると得だが、無理に取ると危ない」配置を使います。
- 安全な位置だけでなく少し危ない位置にも出し、「取りに行くか、安全に避けるか」を判断させます。ただし 👯‍♀️ は置物型の無敵アイテムとして地上付近に配置し、高所やランダムな高低差には出しません。
- 高所アイテムは 1 回ジャンプで届く範囲に制限します。通常配置は `groundY - 76 - random(0〜18)`、高所配置は `groundY - 96 - random(0〜18)` を基準にします。👯‍♀️ は `groundY - 34` を基準に地上付近へ置き、他アイテムより遅い `DANCER_PLACE_SCROLL_MULT` で流して最低 1.5〜2 秒程度は認識しやすくします。
- 取得判定は見た目より少し大きめにします。
- 🔩 と ⚙️ は、🚚を通り過ぎた直後（`player.x + player.w + 12` を超えた時点）で active かつ未取得なら、1個につき 1 回だけ取り逃がしペナルティ -50m を加えます。画面右端まで待たずに、プレイヤーの近くへ `-0.05km` を表示します。事故回数、警戒度、逃走モード、🚓 表示は増やしません。
- 💰 と 👯‍♀️ は取り逃がしても減点しません。
- 👯‍♀️ を取得すると 4 秒間だけ障害物事故に対して無敵になり、障害物に当たっても事故ペナルティ、警戒度上昇、逃走モード移行は発生しません。
- 👯‍♀️ はスコア加算なし、スコア減点なしです。序盤 0.50km 未満では出さず、0.50km 以降に低確率で出します。今回の緊急調整では出現頻度を前回比で約半分にしましたが、取得効果は 4 秒間のまま維持します。描画時は少し大きくし、薄い輪と「無敵」ラベルで認識しやすくします。
- 👯‍♀️ 無敵中でも穴に落ちた場合は即終了します。穴即終了仕様は変更しません。

## 10. ランキング仕様

- ランキングを使います。Supabase URL 未設定時はゲーム本体を止めず「ランキング未設定」と表示します。
- ベストスコア型です。
- ランキング送信値は補正込みスコアのメートル整数であり、HUD の走行距離ではありません。
- 補正込み走行距離のメートル整数が高いほど上位です。計算式は `実走行距離 + アイテム加算 - 事故ペナルティ - 取り逃がしペナルティ` で、表示と送信値は 0 未満にしません。
- 同じ名前のプレイヤーが複数回遊んだ場合、ランキング表示では一番高い記録を使う想定です。
- 上位 10 件を表示します。
- 送信は結果確定後に 1 回だけです。
- 通信失敗時も結果画面、シェア、再挑戦ボタンは壊しません。ランキング表示は「ランキング未設定」「ランキング送信に失敗しました」「ランキングを取得できませんでした」「ランキングなし」を混同しないよう分けます。
- 表示名は `display_name`、`player_name`、`name`、`normalized_name` の順で安全に参照します。
- スコアは `best_score`、`score`、`first_score` の順で参照し、最後は `0` として扱います。表示は必ず `0.00km` 形式で、`NaNkm` を出しません。

## 11. Supabase仕様

`index.html` 上部に次の定数を置きます。

- `SUPABASE_URL`
- `SUPABASE_PUBLISHABLE_KEY`
- `GAME_SLUG`
- `GAME_NAME`
- `CLIENT_VERSION`
- `PUBLIC_URL`

Supabase URL は正式な公開用 URL `https://mlpnjgezrnhdxsxolyzj.supabase.co` を設定しています。Publishable key はユーザー指定の Publishable key のみを使い、secret key、service_role key、管理者用キーは書きません。`SUPABASE_URL` と `SUPABASE_PUBLISHABLE_KEY` の両方がある場合だけ Supabase 接続可能として扱います。

結果詳細の `score`、`runMeters`、`bonusMeters`、`penaltyMeters`、`missPenaltyMeters`、`accidents`、`items`、`elapsedMs`、`finishReason`、`maxDanger` は `resultSnapshot` 内に保持します。`resultSnapshot` を結果表示・ランキング送信の唯一のソースとします。結果画面の大きなスコアは必ず `resultSnapshot.score` の補正込みスコアを表示し、ランキング送信値 `p_score` も同じ値に一致させます。補正込みスコアは `calculateFinalScoreMeters()` で 0 未満にクランプし、`NaNkm`、`Infinitykm`、実走行距離があるのに常時 `0.00km` になる表示を禁止します。結果画面には実走行距離、アイテム加算、事故ペナルティ、取り逃がしペナルティ、事故回数、取得アイテム数、走行時間、終了理由、穴/障害物/対向障害物/👯‍♀️/全アイテムの出現数カウンターを毎回作り直して表示します。ただし、現在確認できている共通 `submit_score` RPC には metadata 引数を送らず、存在しない引数による RPC 失敗を避けます。

RPC は後から直しやすいよう、`sendScoreAfterResult(result)`、`submitScoreToSupabase(payload)`、`fetchBestRanking()`、`fetchPlayStats()`、`renderRanking()` に分離しています。送信は `game_scores` への直接 insert ではなく、共通仕様の `/rest/v1/rpc/submit_score` を呼びます。取得は `/rest/v1/rpc/get_best_score_ranking` と `/rest/v1/rpc/get_game_play_stats` です。Supabase URL または Publishable key が未設定の場合は通信せず「ランキング連携：設定後に有効になります」と表示します。

ランキング送信は、結果確定時にスコアが有限数値であることを確認し、`kiriganaito_pending_scores_v1` の pending queue へ保存してから送信します。pending score は `id`、`displayName`、`gameSlug`、`score`、`clientVersion`、`createdAt`、`attempts`、`lastError` を持ち、localStorage 保存に失敗してもゲーム進行は止めません。送信失敗時は localStorage に未送信スコアを残し、次回表示、ページ初期化、オンライン復帰、フォーカス復帰、表示復帰時に `flushPendingScores()` で再送できるようにします。iPhone Safari で共有、戻る、閉じる、別アプリ移動、通信不安定などが起きても、可能な範囲で結果記録を失わない設計にします。

旧キー `songen_wo_kakeyouka2_pending_scores_v1` は移行専用です。旧キーが存在する場合は `gameSlug === "kiriganaito"` の未送信スコアだけを `kiriganaito_pending_scores_v1` へ安全に移行し、別ゲームの未送信データは移動または削除しません。旧キーに kiriganaito 以外のデータが混在している場合は、その別ゲームのデータだけを旧キーに残します。この移行は保存キー修正のための処理であり、ランキング RPC 仕様、スコア計算、ゲームルールは変更しません。

結果画面にはランキング保存状態を表示し、pending score がある場合は手動再送ボタン「ランキング再送」を表示します。スコア登録の成功・保存待ちとランキング取得失敗は別扱いにし、ランキング取得失敗時は「ランキングを読み込めませんでした」と表示しても、登録成功状態を消しません。ホーム画面と結果画面には小さく `CLIENT_VERSION` を表示し、HTML には `x-client-version` メタも置いて、古い HTML キャッシュの問い合わせ確認に使います。結果内訳のクリアは `textContent = ""` または `replaceChildren()` とし、読み取り専用の `children` へ代入しません。

RPC 引数は共通仕様に合わせて次の形に統一します。

- `/rest/v1/rpc/submit_score`
  - `p_game_slug`: `kiriganaito`
  - `p_display_name`: プレイヤー名
  - `p_score`: 結果確定時の整数メートルスコア
  - `p_client_version`: `kiriganaito-2026-06-26-v2-result-density`
- `/rest/v1/rpc/get_best_score_ranking`
  - `p_game_slug`: `kiriganaito`
  - `p_limit`: `10`
- `/rest/v1/rpc/get_game_play_stats`
  - `p_game_slug`: `kiriganaito`

`game_slug`、`display_name`、`score`、`metadata` のような直接 insert 風 payload は `submit_score` に送りません。ランキング取得も `game_slug`、`limit_count` などの別名引数を同時送信せず、`p_` 付き引数だけを送ります。


## 11.1 ゲーム終了処理

終了は `finishGame(reason)` に集約します。

1. すでに `PLAYING` でなければ何もしません。
2. `mode` を `RESULT` にします。
3. `getFinalRunMeters()` 相当で `run.runMeters`、`run.maxRunMeters`、`run.lastNonZeroRunMeters` の最大値を丸め、穴落ちや表示切替直前の 0 戻りを防いだ最終実走行距離を固定します。
4. `calculateFinalScoreMeters()` で補正込みの `finalScore` と終了理由を固定します。
5. 結果画面用の `resultSnapshot` を唯一の結果ソースとして作成します。
6. 結果画面を表示します。
7. ランキング送信を 1 回だけ開始します。
8. ランキング取得を行います。

結果画面後はプレイヤー更新、障害物更新、アイテム更新、パーティクル更新、穴更新、距離加算、逃走時間更新、当たり判定、タイマー更新を再開しません。`requestAnimationFrame` は 1 本だけにし、`RESULT` では再予約しません。リトライ時は `resetState()` で前回状態を初期化します。

## 19. 実装方針

- HTML/CSS/JavaScript を `index.html` にまとめます。
- JavaScript は、定数、Supabase設定、DOM取得、状態管理、入力処理、ゲーム初期化、ゲーム更新、当たり判定、描画、UI更新、音、結果処理、ランキング送信、ランキング取得、シェア、デバッグ、起動処理のコメント区画に分けます。
- ゲームループは `requestAnimationFrame` 1 本だけにし、重複起動を防ぎます。`loop(ts)` は可変フレーム時間を accumulator に入れ、`update(1/60)` を最大 5 回まで回してから `draw()` を毎フレーム 1 回だけ呼びます。
- DOM の毎フレーム大量更新を避け、HUD は約 0.1 秒ごとに更新します。
- 障害物、アイテム、パーティクルは配列で管理し、上限を超えないようにします。
- ページ非表示時は `PAUSED` 相当にして大きな時間ジャンプを防ぎます。
- Canvas 内部解像度は DPR 2 までに制限します。

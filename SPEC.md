# kiriganaito 仕様書

## 1. ゲーム概要

`kiriganaito` は、左へ走っているように見える 🚚 をタップでジャンプさせ、穴・でこぼこ道・通行人・車両を避けながら補正込み走行距離を伸ばすスマホ向けブラウザゲームです。

- 公開予定 URL: `https://chameleonjp.codeberg.page/kiriganaito/`
- ゲーム slug: `kiriganaito`
- 表示名: `kiriganaito`
- 想定端末: iPhone SE 相当の小さいスマホ縦画面
- プレイ時間目安: 標準 45〜90 秒、上級者は 2〜3 分以上
- 実装形態: `index.html` だけで完結する HTML/CSS/JavaScript 単体ファイル

## 2. 固定仕様

- 外部画像、外部音声、外部フォント、npm、Vite、ビルド環境は使いません。
- ゲーム中に大量に動く要素は Canvas に描画します。
- 障害物、アイテム、パーティクルを DOM 要素として大量配置しません。
- 横スクロールを出しません。
- 結果画面に入ったら、ゲーム更新、敵移動、タイマー、当たり判定、スコア加算を止めます。
- ランキング送信はゲーム終了時に自動で 1 回だけ行います。
- 結果画面にランキング登録ボタンは置きません。
- Supabase で使ってよいキーは Publishable key のみです。
- `public.scores` は使いません。
- secret key、service_role key、管理者用キーは書きません。

## 3. 勝利条件、敗北条件

固定ゴールはありません。1 回のプレイで補正込み走行距離をできるだけ伸ばすことが目的です。

敗北・終了条件は次の 3 つです。

1. 穴に落ちる
   - 即終了。
   - 事故扱いではありません。
   - 無敵時間中でも即終了します。
2. 警戒度が 3 になる
   - 障害物にぶつかるたびに警戒度が 1 上がります。
   - 警戒度 3 で 🚓 に捕まって終了します。
3. リタイアする
   - 上部のリタイアボタンで終了します。
   - その時点の補正込み走行距離を結果として扱います。
   - ランキング送信も 1 回だけ行います。

## 4. 操作方法

- Canvas をタップ、または下部の大きいジャンプボタンをタップしてジャンプします。
- 2 段ジャンプはありません。
- コヨーテタイム: 地面から離れた後 `0.08` 秒までジャンプを受け付けます。
- ジャンプバッファ: 着地直前タップを `0.12` 秒まで保存します。
- 衝突後の障害物判定無敵時間は `0.7` 秒です。
- 入力は `pointerdown` 中心で扱い、タッチ・マウス・ペンをまとめます。
- 結果画面ではジャンプ入力を無効にし、結果画面のボタンだけ反応します。

## 5. スコア計算

スコア名は「補正込み走行距離」です。

- 単位: km
- 表示形式: `0.00km`
- 内部送信値: 整数メートル
- 高い方がよいベストスコア型

計算式:

```text
補正込み走行距離 = 実走行距離 + アイテム加算 - 事故ペナルティ
```

- 事故ペナルティ: 1 回 `-0.20km`、内部値 `-200m`
- 表示値と送信値は 0 未満にしません。
- 逃走中も距離加算は通常のままです。見た目や接近速度だけ 2 倍になります。
- 結果確定後にスコアは変えません。

## 6. 障害物仕様

障害物は Canvas 上で絵文字として描きます。内部では `direction` で進行方向を管理します。

| 種類 | 速度 |
| --- | ---: |
| 🛵 | 3.6 |
| 🚗 | 3.8 |
| 🏃🏻 | 2.0 |
| 🚴 | 3.0 |
| 🚶 | 1.0 |

- 左向き障害物: 🚚 と同じ方向に動く相手で、🚚 が後ろから追いつく見た目です。
- 右向き障害物: 🚚 と逆方向へ動く相手で、正面から向かってくるため接近が速い見た目です。描画時は左右反転します。
- 同じ障害物で連続減点しないよう、衝突した障害物は非アクティブ化します。
- 距離に応じて、🚶、🏃🏻、🚴、🛵、🚗、対向障害物の順で増やします。


## 6.1 出現ロジックと難易度上昇

出現は毎フレーム乱数だけに頼らず、距離ベースの予定値で管理します。

- `nextObstacleAt`、`nextHoleAt`、`nextItemAt` で次の出現距離を管理します。
- `lastDangerAt`、`lastHoleAt`、`lastObstacleAt`、`lastOncomingAt` で直近の危険配置を記録します。
- 逃走中は見た目速度が 2 倍になるため、次の穴・障害物予定距離に追加の余白を入れます。

| 距離 | 内容 |
| --- | --- |
| 0.00〜0.50km | 練習区間。短い穴、🚶中心。対向障害物なし。 |
| 0.50〜1.50km | 🏃🏻、🚴を追加。穴は少し増え、対向はごく控えめ。 |
| 1.50〜3.00km | 🛵、🚗を追加。左向き中心。アイテムを少し危険な高さにも置く。 |
| 3.00〜5.00km | 対向障害物を本格追加。ただし連続対向は禁止。 |
| 5.00km以降 | 穴、対向障害物、アイテム誘導を組み合わせるが、避けられない配置は禁止。 |

最低間隔の実装値は次の通りです。単位は km です。

| 制限 | 値 |
| --- | ---: |
| 障害物直後に穴を出さない距離 | `0.16` |
| 穴直後に障害物を出さない距離 | `0.24` |
| 障害物同士の最低距離 | `0.18` |
| 対向障害物後の安全距離 | `0.36` |
| アイテムと障害物の最低距離 | `0.10` |
| アイテムと穴の最低距離 | `0.08` |

## 7. 事故・逃走モード仕様

障害物にぶつかると事故扱いになります。

- 補正込み走行距離から `-0.20km`、内部値 `-200m`
- 画面上部で 🚨 が光ります。
- 🚓 が 🚚 を追いかけます。
- 15 秒間、逃走モードになります。
- 逃走モード中は見た目と障害物接近速度を 2 倍にします。
- 逃走モード中は背景に「逃げろ」を左から右へ流します。
- 警戒度を 1 上げます。
- 逃走中に再事故した場合も同じ処理を行い、逃走残り時間は 15 秒に戻します。
- 警戒度が 3 になったら即終了します。
- 逃走が終わっても警戒度は下がりません。

## 8. 道、穴、でこぼこ仕様

- 穴はジャンプで越える必要があります。
- 穴に落ちたら即終了します。
- 穴幅は 1 回ジャンプで越えられる範囲に制限します。
- 2 段ジャンプが必要な穴幅は出しません。
- でこぼこは直接ダメージにせず、地面の高さの軽い上下と見た目演出に使います。
- 見た目が荒れていても当たり判定の地面はなだらかにします。
- 穴直後に避けられない障害物を置かず、最低間隔を持たせます。

## 9. アイテム仕様

拾うと補正込み走行距離に加算します。

| アイテム | 表示加算 | 内部値 |
| --- | ---: | ---: |
| 🪙 | +0.10km | +100m |
| 🔩 | +0.03km | +30m |
| ⚙️ | +0.07km | +70m |

- 安全な位置だけでなく少し危ない位置にも出し、「取りに行くか、安全に避けるか」を判断させます。
- 取得判定は見た目より少し大きめにします。


## 9.1 当たり判定定数

当たり判定は `HIT` 定数にまとめ、調整しやすくしています。

| 定数 | 値 | 意味 |
| --- | ---: | --- |
| `PLAYER_SHRINK_X` | `8` | 🚚 の左右判定縮小量 |
| `PLAYER_SHRINK_TOP` | `8` | 🚚 の上側判定縮小量 |
| `PLAYER_SHRINK_BOTTOM` | `6` | 🚚 の下側判定縮小量 |
| `OBSTACLE_SHRINK` | `5` | 障害物判定縮小量 |
| `ITEM_EXPAND` | `10` | アイテム取得判定拡大量 |
| `HOLE_MARGIN_X` | `7` | 穴判定の左右余白 |
| `HOLE_FALL_Y` | `2` | 穴落下判定の下方向条件 |
| `INVINCIBLE_SEC` | `0.7` | 衝突後の障害物判定無敵時間 |
| `COYOTE_SEC` | `0.08` | コヨーテタイム |
| `JUMP_BUFFER_SEC` | `0.12` | ジャンプバッファ |

🚚 と障害物は見た目より少し小さく、アイテムは少し取りやすく、穴は見た目と大きくズレない範囲にしています。

## 10. ランキング仕様

- ランキングを使います。Supabase URL 未設定時はゲーム本体を止めず「ランキング未設定」と表示します。
- ベストスコア型です。
- 補正込み走行距離のメートル整数が高いほど上位です。
- 同じ名前のプレイヤーが複数回遊んだ場合、ランキング表示では一番高い記録を使う想定です。
- 上位 10 件を表示します。
- 送信は結果確定後に 1 回だけです。
- 通信失敗時も結果画面、シェア、再挑戦ボタンは壊しません。ランキング表示は「ランキング未設定」「ランキング送信に失敗しました」「ランキングを取得できませんでした」「ランキングなし」を混同しないよう分けます。
- 表示名は `display_name`、`player_name`、`name`、`normalized_name` の順で安全に参照します。
- スコアは `best_score`、`score`、`first_score` の順で参照し、最後は `0` として扱います。表示は必ず `0.00km` 形式で、`NaNkm` を出しません。

## 11. Supabase仕様

`index.html` 上部に次の定数を置きます。

- `SUPABASE_URL`
- `SUPABASE_PUBLISHABLE_KEY`
- `GAME_SLUG`
- `GAME_NAME`
- `CLIENT_VERSION`
- `PUBLIC_URL`

Supabase URL は正式な公開用 URL `https://mlpnjgezrnhdxsxolyzj.supabase.co` を設定しています。Publishable key はユーザー指定の Publishable key のみを使い、secret key、service_role key、管理者用キーは書きません。`SUPABASE_URL` と `SUPABASE_PUBLISHABLE_KEY` の両方がある場合だけ Supabase 接続可能として扱います。

結果詳細の `runMeters`、`bonusMeters`、`penaltyMeters`、`accidents`、`items`、`elapsedMs`、`finishReason`、`maxDanger` は `resultSnapshot` 内に保持します。ただし、現在確認できている共通 `submit_score` RPC には metadata 引数を送らず、存在しない引数による RPC 失敗を避けます。

RPC は後から直しやすいよう、`submitScoreOnce()`、`submitScoreToSupabase(payload)`、`fetchBestRanking()`、`fetchPlayStats()`、`renderRanking()` に分離しています。送信は `game_scores` への直接 insert ではなく、共通仕様の `/rest/v1/rpc/submit_score` を呼びます。取得は `/rest/v1/rpc/get_best_score_ranking` と `/rest/v1/rpc/get_game_play_stats` です。Supabase URL または Publishable key が未設定の場合は通信せず「ランキング未設定」と表示します。

RPC 引数は共通仕様に合わせて次の形に統一します。

- `/rest/v1/rpc/submit_score`
  - `p_game_slug`: `kiriganaito`
  - `p_display_name`: プレイヤー名
  - `p_score`: 結果確定時の整数メートルスコア
  - `p_client_version`: `kiriganaito-2026-06-25-v1`
- `/rest/v1/rpc/get_best_score_ranking`
  - `p_game_slug`: `kiriganaito`
  - `p_limit`: `10`
- `/rest/v1/rpc/get_game_play_stats`
  - `p_game_slug`: `kiriganaito`

`game_slug`、`display_name`、`score`、`metadata` のような直接 insert 風 payload は `submit_score` に送りません。ランキング取得も `game_slug`、`limit_count` などの別名引数を同時送信せず、`p_` 付き引数だけを送ります。


## 11.1 ゲーム終了処理

終了は `finishGame(reason)` に集約します。

1. すでに `PLAYING` でなければ何もしません。
2. `mode` を `RESULT` にします。
3. `finalScore` と終了理由を固定します。
4. 結果画面用の `resultSnapshot` を作成します。
5. 結果画面を表示します。
6. ランキング送信を 1 回だけ開始します。
7. ランキング取得を行います。

結果画面後はプレイヤー更新、障害物更新、アイテム更新、パーティクル更新、穴更新、距離加算、逃走時間更新、当たり判定、タイマー更新を再開しません。`requestAnimationFrame` は 1 本だけにし、`RESULT` では再予約しません。リトライ時は `resetState()` で前回状態を初期化します。

## 11.2 今回変更しない仕様

今回の更新は Supabase 実接続、RPC 引数の統一、ランキング表示分岐、仕様書更新に限定します。ゲームルール、スコア計算、出現ロジック、当たり判定、ジャンプ数値、画面演出は変更しません。

## 12. 名前入力仕様

- ランキングを使うため名前入力は必須です。
- 名前は 1〜10 文字です。
- 前後の空白を削ります。
- 空欄では開始できません。
- 一度入力した名前は `localStorage` に保存してよいです。
- 公式ランキングのスコア本体は `localStorage` に保存しません。
- 表示は `textContent` 相当の安全な出し方にします。
- ホームから開始するとき、名前が未入力なら名前入力へ進みます。

## 13. シェア仕様

ホームと結果にシェアボタンを置きます。

ホームのシェア文:

```text
kiriganaito

https://chameleonjp.codeberg.page/kiriganaito/
```

結果のシェア文:

```text
kiriganaito
結果：{scoreKm}km
事故：{accidents}回 / 終了理由：{reason}
https://chameleonjp.codeberg.page/kiriganaito/
```

Web Share API が使える場合は使い、使えない場合はクリップボードコピーにします。コピーも失敗した場合は短いメッセージだけ出します。

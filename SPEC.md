# kiriganaito 仕様書

## 1. ゲーム概要

`kiriganaito` は、左へ走っているように見える 🚚 をタップでジャンプさせ、穴・でこぼこ道・通行人・車両を避けながら補正込み走行距離を伸ばすスマホ向けブラウザゲームです。

- 公開予定 URL: `https://chameleonjp.codeberg.page/kiriganaito/`
- ゲーム slug: `kiriganaito`
- 表示名: `kiriganaito`
- 想定端末: iPhone SE 相当の小さいスマホ縦画面
- プレイ時間目安: 標準 45〜90 秒、上級者は 2〜3 分以上
- 実装形態: `index.html` だけで完結する HTML/CSS/JavaScript 単体ファイル

## 2. 固定仕様

- 外部画像、外部音声、外部フォント、npm、Vite、ビルド環境は使いません。
- ゲーム中に大量に動く要素は Canvas に描画します。
- 障害物、アイテム、パーティクルを DOM 要素として大量配置しません。
- 横スクロールを出しません。
- 結果画面に入ったら、ゲーム更新、敵移動、タイマー、当たり判定、スコア加算を止めます。
- ランキング送信はゲーム終了時に自動で 1 回だけ行います。
- 結果画面にランキング登録ボタンは置きません。
- Supabase で使ってよいキーは Publishable key のみです。
- `public.scores` は使いません。
- secret key、service_role key、管理者用キーは書きません。

## 3. 勝利条件、敗北条件

固定ゴールはありません。1 回のプレイで補正込み走行距離をできるだけ伸ばすことが目的です。

敗北・終了条件は次の 3 つです。

1. 穴に落ちる
   - 即終了。
   - 事故扱いではありません。
   - 無敵時間中でも即終了します。
2. 警戒度が 3 になる
   - 障害物にぶつかるたびに警戒度が 1 上がります。
   - 警戒度 3 で 🚓 に捕まって終了します。
3. リタイアする
   - 上部のリタイアボタンで終了します。
   - その時点の補正込み走行距離を結果として扱います。
   - ランキング送信も 1 回だけ行います。

## 4. 操作方法

- Canvas をタップ、または下部の大きいジャンプボタンをタップしてジャンプします。
- 2 段ジャンプはありません。
- コヨーテタイム: 地面から離れた後 `0.08` 秒までジャンプを受け付けます。
- ジャンプバッファ: 着地直前タップを `0.12` 秒まで保存します。
- 衝突後の障害物判定無敵時間は `0.7` 秒です。
- 入力は `pointerdown` 中心で扱い、タッチ・マウス・ペンをまとめます。
- 結果画面ではジャンプ入力を無効にし、結果画面のボタンだけ反応します。

## 5. スコア計算

スコア名は「補正込み走行距離」です。

- 単位: km
- 表示形式: `0.00km`
- 内部送信値: 整数メートル
- 高い方がよいベストスコア型

計算式:

```text
補正込み走行距離 = 実走行距離 + アイテム加算 - 事故ペナルティ
```

- 事故ペナルティ: 1 回 `-0.20km`、内部値 `-200m`
- 表示値と送信値は 0 未満にしません。
- 逃走中も距離加算は通常のままです。見た目や接近速度だけ 2 倍になります。
- 結果確定後にスコアは変えません。

## 6. 障害物仕様

障害物は Canvas 上で絵文字として描きます。内部では `direction` で進行方向を管理します。

| 種類 | 速度 |
| --- | ---: |
| 🛵 | 3.6 |
| 🚗 | 3.8 |
| 🏃🏻 | 2.0 |
| 🚴 | 3.0 |
| 🚶 | 1.0 |

- 左向き障害物: 🚚 と同じ方向に動く相手で、🚚 が後ろから追いつく見た目です。
- 右向き障害物: 🚚 と逆方向へ動く相手で、正面から向かってくるため接近が速い見た目です。描画時は左右反転します。
- 同じ障害物で連続減点しないよう、衝突した障害物は非アクティブ化します。
- 距離に応じて、🚶、🏃🏻、🚴、🛵、🚗、対向障害物の順で増やします。

## 7. 事故・逃走モード仕様

障害物にぶつかると事故扱いになります。

- 補正込み走行距離から `-0.20km`、内部値 `-200m`
- 画面上部で 🚨 が光ります。
- 🚓 が 🚚 を追いかけます。
- 15 秒間、逃走モードになります。
- 逃走モード中は見た目と障害物接近速度を 2 倍にします。
- 逃走モード中は背景に「逃げろ」を左から右へ流します。
- 警戒度を 1 上げます。
- 逃走中に再事故した場合も同じ処理を行い、逃走残り時間は 15 秒に戻します。
- 警戒度が 3 になったら即終了します。
- 逃走が終わっても警戒度は下がりません。

## 8. 道、穴、でこぼこ仕様

- 穴はジャンプで越える必要があります。
- 穴に落ちたら即終了します。
- 穴幅は 1 回ジャンプで越えられる範囲に制限します。
- 2 段ジャンプが必要な穴幅は出しません。
- でこぼこは直接ダメージにせず、地面の高さの軽い上下と見た目演出に使います。
- 見た目が荒れていても当たり判定の地面はなだらかにします。
- 穴直後に避けられない障害物を置かず、最低間隔を持たせます。

## 9. アイテム仕様

拾うと補正込み走行距離に加算します。

| アイテム | 表示加算 | 内部値 |
| --- | ---: | ---: |
| 🪙 | +0.10km | +100m |
| 🔩 | +0.03km | +30m |
| ⚙️ | +0.07km | +70m |

- 安全な位置だけでなく少し危ない位置にも出し、「取りに行くか、安全に避けるか」を判断させます。
- 取得判定は見た目より少し大きめにします。

## 10. ランキング仕様

- ランキングを使います。
- ベストスコア型です。
- 補正込み走行距離のメートル整数が高いほど上位です。
- 同じ名前のプレイヤーが複数回遊んだ場合、ランキング表示では一番高い記録を使う想定です。
- 上位 10 件を表示します。
- 送信は結果確定後に 1 回だけです。
- 通信失敗時も結果画面、シェア、再挑戦ボタンは壊しません。

## 11. Supabase仕様

`index.html` 上部に次の定数を置きます。

- `SUPABASE_URL`
- `SUPABASE_PUBLISHABLE_KEY`
- `GAME_SLUG`
- `GAME_NAME`
- `PUBLIC_URL`

今回のリポジトリ内には既存の共通ランキング処理や Supabase URL を確認できなかったため、URL は空文字、Publishable key はユーザー指定値を入れています。URL が空の間は通信せず「ランキング未設定」と表示します。

送信メタ情報:

- `runMeters`
- `bonusMeters`
- `penaltyMeters`
- `accidents`
- `items`
- `elapsedMs`
- `finishReason`
- `maxDanger`

RPC は後から直しやすいよう、`submitScore`、`fetchBestRanking`、`fetchPlayStats` に分離しています。想定 RPC は `get_best_score_ranking` と `get_game_play_stats` です。不明な RPC 引数名は仮実装の中に閉じ込めています。

## 12. 名前入力仕様

- ランキングを使うため名前入力は必須です。
- 名前は 1〜10 文字です。
- 前後の空白を削ります。
- 空欄では開始できません。
- 一度入力した名前は `localStorage` に保存してよいです。
- 公式ランキングのスコア本体は `localStorage` に保存しません。
- 表示は `textContent` 相当の安全な出し方にします。
- ホームから開始するとき、名前が未入力なら名前入力へ進みます。

## 13. シェア仕様

ホームと結果にシェアボタンを置きます。

ホームのシェア文:

```text
kiriganaito
逆走する🚚で穴と障害物をジャンプ回避。事故ると🚓から15秒逃走！
https://chameleonjp.codeberg.page/kiriganaito/
```

結果のシェア文:

```text
kiriganaito
結果：{scoreKm}km
事故：{accidents}回 / 終了理由：{reason}
https://chameleonjp.codeberg.page/kiriganaito/
```

Web Share API が使える場合は使い、使えない場合はクリップボードコピーにします。コピーも失敗した場合は短いメッセージだけ出します。

## 14. 画面構成

- HOME: ホーム画面
- RULES: ルール説明
- NAME: 名前入力
- READY: 開始直前状態
- PLAYING: ゲーム中
- PAUSED: ページ非表示などの一時停止相当
- RESULT: 結果画面
- ERROR: エラー表示

ホーム画面にはゲーム名、一言説明、スタート、ルール、シェア、ランキング欄、「他のゲームで遊ぶ」リンクを置きます。

ゲーム画面には上部 HUD、Canvas、下部ジャンプボタンを置きます。ゲーム盤面と操作ボタンは重ねません。

結果画面には終了理由、最終スコア、内訳、事故回数、ランキング送信状態、ランキング、結果シェア、もう一度、ホーム、「他のゲームで遊ぶ」を置きます。

## 15. 性能予算

- 目標 fps: 60fps、難しい端末でも 30fps 安定
- 動く障害物: 最大 18 体
- 弾: 0 発
- アイテム: 最大 12 個
- 道の区間: 最大 34 区間
- パーティクル総数: 最大 96 個
- 「逃げろ」文字パーティクル: 最大 18 個
- 事故・火花・拾得エフェクト: 合計 78 個まで
- Canvas で 1 フレームに描く要素: 約 170 個まで
- ゲーム中に動かす DOM: 0 個
- ゲーム中に更新する DOM テキスト: 最大 6 か所
- ランキング送信: ゲーム終了時に 1 回だけ
- ランキング取得: ホームで最大 1 回、結果で最大 1 回
- 統計取得: 結果で最大 1 回
- `devicePixelRatio` 上限: 2

## 18. 実装方針

- HTML/CSS/JavaScript を `index.html` にまとめます。
- JavaScript は、定数、Supabase設定、DOM取得、状態管理、入力処理、ゲーム初期化、ゲーム更新、当たり判定、描画、UI更新、音、結果処理、ランキング送信、ランキング取得、シェア、デバッグ、起動処理のコメント区画に分けます。
- ゲームループは `requestAnimationFrame` 1 本だけにし、重複起動を防ぎます。
- DOM の毎フレーム大量更新を避け、HUD は約 0.1 秒ごとに更新します。
- 障害物、アイテム、パーティクルは配列で管理し、上限を超えないようにします。
- ページ非表示時は `PAUSED` 相当にして大きな時間ジャンプを防ぎます。
- Canvas 内部解像度は DPR 2 までに制限します。

# P7 UI・デザイン・説明の仕上げ契約

- 対象版: `kiriganaito-2026-07-21-v24-ui-finish`
- 対象段階: `P7`
- 基準main: `47a58cc5b50231b9a4595b70c54e12bb00d9c0f0`
- 目的: 初見の理解、プレイ中の判別、結果画面の情報優先順位をiPhone SE相当で仕上げる

## 1. 今回の作業単位

P7では次の3点だけを変更する。

1. 結果画面では最終記録と主要4項目、シェア、再戦を先に見せ、詳細統計と診断を閉じる。
2. 320px幅・短い縦画面でも、HUD、ゲーム画面、ジャンプ、状態表示を収めやすくする。
3. 穴、吊り下げバー、対向障害物、逃走状態を色だけでなく文字・形・方向でも判別可能にする。

スコア、出現密度、速度、TTC、当たり判定、SpawnDirector、ランキングRPC、Supabase設定は変更しない。

## 2. ホームと説明

- ホーム上部に「タップ=1段ジャンプ」「穴=落ちると終了」「吊りバー=跳ばずに通過」を短く表示する。
- 従来の詳しいルールを残し、初見向け要点と詳細説明を分離する。
- 詳細ランキングは初期状態で閉じたままにする。

## 3. ゲーム画面

- `320x568`を短い縦画面の基準にする。
- `100dvh`からsafe areaを差し引き、ゲーム領域をflexで縮小可能にする。
- 高さ640px以下ではCanvas、HUD、ジャンプボタン、状態欄を圧縮する。
- ジャンプボタンの最小高さは短い画面でも60pxを維持する。
- 横スクロールを発生させない。

危険の表示契約:

- 穴: 明るい左右端、警告記号、「穴」ラベル。
- 吊り下げバー: 吊り線、地面への影、「跳ばない」ラベル。
- 対向障害物: 左側速度線、右向き矢印を含む「→ 対向」ラベル。
- 逃走: 画面全面を覆う赤演出をやめ、左右端の警告帯と「🚨 逃走中」ラベルにする。

## 4. 結果画面

初期表示の優先順:

1. 終了理由と短いコメント
2. 最終記録
3. 実走行距離、アイテム取得数/出現数、取得率、事故回数
4. 結果シェア、もう一度
5. ランキング送信状態
6. 閉じた「記録の詳しい内訳・プレイ統計」
7. 閉じた詳細ランキング

詳細統計内の内部診断はさらに「詳細診断を表示」を押すまで隠す。通常プレイヤーへSpawnDirector、TTC、retry、内部カウンターを常時見せない。結果表示時には詳細とランキングを閉じ、ページ上端へ戻す。

## 5. 自動合格条件

`node tests/p7-ui-finish-regression.js`で次を確認する。

- v24のversion表記が一致する。
- ホーム要点3項目が存在する。
- 結果要点4項目と主要2操作が詳細より前にある。
- 結果詳細とランキングが初期状態で閉じる。
- 診断行が初期非表示で、操作後に表示できる。
- 穴、空中危険、対向危険の文字ラベルがCanvasへ実際に描かれる。
- 320px横幅のoverflow防止と短い縦画面用CSSが存在する。
- console error / warningが0。

さらにP1〜P6、v22 UI、150km、release comprehensiveを全回帰する。

## 6. 実ブラウザ確認

320x568のブラウザでホーム、ゲーム、結果を通し、次を確認する。

- `scrollWidth <= innerWidth`
- HUD、Canvas、ジャンプボタン、状態表示が横にはみ出さない
- 結果の最終記録、結果シェア、もう一度を容易に見つけられる
- 結果詳細が閉じている
- JavaScriptエラー表示がない

# kiriganaito 実機フィードバックUI修正契約

- 対象: P5 main反映後の実機確認指摘
- 基準main: `343f576d375bdf526221f36556d61308e695af4d`
- 予定版: `kiriganaito-2026-07-20-v22-device-feedback-ui`
- 作業ブランチ: `ai/kiriganaito-v22-device-feedback-ui`
- P6スコア最終固定より前に実施する独立修正

## 1. 実機指摘

1. 地面の波打ちで画面酔いする
2. 詳細ランキングを初期表示したくない
3. ホーム画面の情報優先順位と配置を改善したい
4. 👯‍♀️の無敵時間を倍へ延長したい
5. トラックの描写を一回り小さくしたい
6. ルール説明の情報が不足している
7. コンセプト文言を次へ統一したい

```text
落とした積荷と落ちてるお金を拾い集めよう
```

## 2. 地面

`getGroundY()`は常に`groundY`を返します。

```text
物理地面: 水平
描画輪郭: 水平
トラックの接地位置: 水平
障害物・アイテム・穴の基準Y: 水平
```

削除するもの:

- `Math.sin()`による3px上下動
- `visualScroll`を地面Yへ反映する処理

維持するもの:

- 穴判定
- 高速時の穴すり抜け防止
- ジャンプ、重力、着地判定
- 道路の横方向スクロール表現

## 3. 詳細ランキング

ホームと結果画面のランキングをネイティブ`details`へ入れます。

```text
初期状態: 閉じる
summary: 詳細ランキングを見る
open属性: 付けない
```

ランキングRPC、取得件数、並び順、表示値、Supabase設定は変更しません。

ホームランキングは初めて展開した時に取得します。結果ランキングは従来の登録処理を維持し、表示領域だけを閉じます。

## 4. ホーム画面

初期画面の優先順位を次へ整理します。

1. タイトル
2. コンセプト
3. ゲーム開始
4. ルール説明 / ゲームシェア
5. 名前変更
6. 折りたたみ詳細ランキング
7. その他のゲーム

コンセプト本文は完全一致で表示します。

```text
落とした積荷と落ちてるお金を拾い集めよう
```

iPhone SE幅でも主要操作が上部に収まり、不要な上余白を作らないことを条件とします。

## 5. 無敵時間

👯‍♀️無敵を実プレイ時間4秒から8秒へ変更します。

```text
取得直後: 8.000秒
wall-clockのみ進行: 減算なし
PLAYING固定step 240回: 約4秒残る
PLAYING固定step 480回: 0秒
```

維持するもの:

- 障害物事故だけを防止
- 穴は無敵中でも即終了
- 無敵終了後0.14kmのLARGE hole抑制
- 2〜3個の障害物提示契約
- 「無敵で突破!」表示

表示、ルール、ITEM_TYPES、P5契約、P5テストを8秒へ同期します。

## 6. トラック描画

物理サイズと当たり判定は変更しません。

```text
player.w: 42px維持
player.h: 34px維持
表示フォント: 42px → 36px
```

影と無敵オーラだけ表示サイズへ合わせて縮小します。ジャンプ軌道、穴判定、接触判定は変えません。

## 7. ルール説明

一つの長い文章を、次の区分へ整理します。

- ゲームの目的
- 操作
- 拾うもの
- 避けるもの
- 事故と逃走
- 👯‍♀️無敵
- スコア

必須情報:

```text
💰 +0.10km
🔩 +0.03km / 取り逃がし -0.05km
⚙️ +0.07km / 取り逃がし -0.05km
1km未満の事故 -0.10km
1km以降の事故 -0.20km
事故で警戒度+1
警戒度3で終了
穴は常に即終了
吊り下げバーはジャンプしない
逃走15秒、見た目速度2倍、距離加算は通常
👯‍♀️は実プレイ時間8秒、障害物のみ無敵
```

## 8. コンセプト文言の適用先

- ホーム画面
- ルール冒頭
- ホームのゲームシェア文
- 仕様書

結果シェアはプレイ結果を優先し、既存の結果値・URLを維持します。

## 9. 変更禁止

- スコア式
- アイテムの点数
- ペナルティ計算
- ランキング送信値
- Supabase URL / key / RPC
- pending score key
- P2出現密度
- P3判断パターン
- P4吊り下げバーの当たり判定
- P5逃走15秒のイベント構成
- ジャンプ力、重力、固定タイムステップ
- playerの物理サイズ

## 10. 自動合格条件

```text
getGroundY(-200) = getGroundY(0) = getGroundY(200) = groundY
地面YがvisualScrollで変化しない
home/resultのランキングdetailsにopen属性なし
コンセプト完全一致
ホーム主要ボタンのラベルと順序が成立
ルール必須情報をすべて含む
無敵初期値8秒
wall-clockだけでは8秒維持
240固定step後に約4秒
480固定step後に0秒
player物理42x34維持
トラック表示36px
P1〜P5回帰PASS
progressive autoplay PASS
150km endurance PASS
release criticalIssues 0
console error / warning 0
Supabase本番送信0
```

## 11. 実機再確認

- 地面が水平で画面酔いが軽減したか
- トラックが小さすぎず、当たり判定との違和感がないか
- ホームでゲーム開始がすぐ見つかるか
- 詳細ランキングが初期状態で閉じているか
- ルールだけで空中障害物と無敵の違いを理解できるか
- 8秒無敵が長すぎず、取得価値を感じるか

実機再確認が終わるまでDraftを維持します。

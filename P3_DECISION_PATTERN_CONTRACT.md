# kiriganaito P3 判断パターン契約

- 段階: `COMPLETION_PLAN_v1.md` P3
- 基準実装: `kiriganaito-2026-07-18-v18-effective-density`
- 予定版: `kiriganaito-2026-07-19-v19-decision-patterns`
- 作業ブランチ: `ai/kiriganaito-v19-decision-patterns`

## 1. 目的

単体のランダム出現だけでなく、2〜3個の既存オブジェクトからなる短い判断シーケンスを導入します。

この段階では新しい障害物、新しいアイテム、新しい操作は追加しません。既存の地上障害物、対向障害物、穴、加点アイテム、👯‍♀️だけを使用します。

## 2. 記号

| 記号 | 意味 |
| --- | --- |
| `G` | 通常方向の地上障害物 |
| `O` | 対向障害物 |
| `A` | 空中の加点アイテム |
| `H` | 穴 |
| `P` | 👯‍♀️パワーアップ |
| `S` | オブジェクトを置かない安全距離 |

## 3. 初期パターン

| ID | 構成 | 解禁 | 意図 |
| --- | --- | --- | --- |
| `G_S_H` | `G → S → H` | 0km | 障害物を越え、着地後に次の穴を読む |
| `H_S_G` | `H → S → G` | 0km | 穴を越え、着地後に地上障害物を読む |
| `G_A` | `G + 上空A` | 0km | 障害物回避と報酬取得を同じジャンプへまとめる |
| `H_A` | `H + 穴上A` | 0km | 穴回避のジャンプへ報酬選択を加える |
| `O_S_H` | `O → S → H` | 1km | 反射回避後に次の穴を読む |
| `H_S_O` | `H → S → O` | 1km | 着地猶予後に対向障害物へ反応する |
| `G_A_H` | `G → A → H` | 1km | 障害物、報酬、穴の3段判断 |
| `P_G_G` | `P → G → G` | 2kmかつ👯‍♀️出現可能時 | 無敵を取るかに依存せず、2障害物を個別回避できる構成 |

## 4. 安全契約

- 同一時刻に複数の危険物を重ねない
- 同時配置を許可するのは「危険1個 + 報酬1個」だけ
- `G → H` は `SPAWN.HOLE_AFTER_OBSTACLE` 以上を空ける
- `H → G` は `SPAWN.OBSTACLE_AFTER_HOLE` 以上を空ける
- `H → O` は `SPAWN.ONCOMING_AFTER_HOLE_SAFE_GAP` 以上を空ける
- `O → H` は0.18km以上を空ける
- 穴後の `G` または `O` がパターンに含まれる場合、そのステップが穴間必須障害物を担当する
- パターンの生成でも既存TTC、安全距離、画面混雑判定を通す
- パターンが失敗または期限超過した場合は中断し、通常のSpawnDirectorへ戻す
- パターン中に逃走または無敵が始まった場合は中断する
- パターン成立のために当たり判定、速度、穴幅、ジャンプ性能を変更しない

## 5. 選択契約

- 0〜1kmは4種類の学習用パターンだけを選ぶ
- 1km以降に複合パターンを解禁する
- 2km以降かつ👯‍♀️出現可能時だけ `P_G_G` を選べる
- 利用可能パターンをシャッフルバッグで巡回する
- 同じパターン3連続を禁止する
- 10,000回選択時に単一パターンが25%を超えない

## 6. 通常出現との関係

パターンが有効な短い区間では、通常の穴、通常地上障害物、通常対向障害物、通常アイテムを一時保留します。

次は保留しません。

- 2km以内の早期対向障害物保証
- 穴間必須障害物。ただしパターン自身の後続 `G/O` が担当する場合を除く
- 無敵中の強制障害物
- 逃走固有処理

パターン終了後は既存P2予約制御へ戻します。

## 7. 診断値

次を `run` と結果スナップショットへ保持します。

- `patternStartedCount`
- `patternCompletedCount`
- `patternAbortedCount`
- `patternStepResolvedCount`
- `patternStepSkippedCount`
- `patternMaxConsecutiveSame`
- `patternCounts`
- `patternAbortReasons`
- `activePatternName`

生成物には次を付けます。

- `patternId`
- `patternName`
- `patternStepIndex`
- `patternSymbol`

## 8. 合格条件

- パターン定義の安全検査エラー0
- 10,000回選択で同一パターン3連続0
- 10,000回選択で単一パターン比率25%以下
- 0〜1kmで複合パターン選択0
- 1km以降で複合パターンが実際に提示される
- P1固定seedで `spawnSource=pattern` の実効出現が確認できる
- P2の30固定seedゲートを維持
- 最大同時危険数2以下
- 回避不能対向障害物0
- 穴だけ連続0
- queue overflow 0
- SpawnDirector整合式成立
- 既存総合テスト、150km耐久、progressive autoplay成功

## 9. 変更しないもの

- ジャンプ、重力、固定タイムステップ
- 穴幅と穴落ち判定
- 障害物速度
- TTC最低値
- スコアとペナルティ
- ランキング、Supabase、RPC、pending key
- 通常UIとCanvas描画
- 逃走15秒と無敵4秒の完成仕様。これらの専用調整はP5で行う


## 10. 自然抽選と単体検証の区分

- `H_S_O` と `O_S_H` は安全距離上、P2の最大穴間隔0.30kmと同時に常用できないため、定義・10,000回選択・単体実行の検証対象として保持する。
- 自然走行では `G_S_H / H_S_G / G_A / H_A / G_A_H / P_G_G` を使用する。
- 自然パターンは、既に安全に生成された通常または穴間必須の第1オブジェクトを採用し、その後続だけを予約する。
- 150km対向障害物の回帰下限は、P2基準301個に対する約3.7%の許容差として290個とする。
- 個数下限を満たしても、最短TTC、2km以内保証、回避不能0の条件は一切緩和しない。
